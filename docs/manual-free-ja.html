Rapls AI Chatbot マニュアル（Free版）

WordPress 用 AI チャットボットプラグインの全設定を詳しく解説します。
バージョン: 1.5.0 | PHP 7.4+ | WordPress 5.8+

AI 設定

AI プロバイダー

チャットボットが使用する AI サービスを選択します。各プロバイダーごとに API キーの取得とモデルの選択が必要です。

選択肢説明
OpenAI デフォルトGPT-4o、GPT-4 Turbo、GPT-3.5 Turbo など。最も広く利用されている AI プロバイダーです。
Claude (Anthropic)Claude Opus、Sonnet、Haiku モデル。長文の理解と安全性に優れています。
Gemini (Google)Gemini 2.0 Flash、1.5 Pro など。マルチモーダル対応に強みがあります。
OpenRouter100以上のモデルに統一 API でアクセスできるゲートウェイサービスです。
ヒント: プロバイダーを変更しても過去の会話履歴は保持されます。ただし、モデルが異なるため応答のスタイルが変わる場合があります。

OpenAI 設定

OpenAI の API キーとモデルを設定します。API キーは OpenAI ダッシュボードから取得できます。

API キー: sk- で始まるキーを入力してください。保存時に暗号化されます。「接続テスト」ボタンでキーの有効性を確認できます。

モデル選択:

モデル特徴推奨用途
gpt-4o-mini デフォルト高速・低コスト。GPT-4o の軽量版一般的なカスタマーサポート
gpt-4o高性能マルチモーダル。画像理解対応画像を扱う高品質な応答が必要な場合
gpt-4-turboGPT-4 の高速版。128K コンテキスト長い会話や大量の知識ベースがある場合
gpt-3.5-turbo最も低コストコスト重視の場合
o1 / o1-mini推論特化モデル。思考プロセスを表示複雑な問題解決
o3-mini最新の推論モデル高度な推論タスク
gpt-4.5-previewGPT-4.5 プレビュー版最新機能の試用
gpt-5最新世代。トークン自動乗算機能あり最高品質の応答
GPT-5 トークン乗算: GPT-5 モデル使用時は、設定した最大トークン数に自動的に乗算係数（デフォルト: 4倍）が適用されます。これは GPT-5 の推論トークン消費に対応するためです。

Claude (Anthropic) 設定

Anthropic の API キーとモデルを設定します。API キーは Anthropic コンソールから取得できます。

モデル選択:

モデル特徴
claude-sonnet-4-20250514 デフォルト最新の Sonnet。速度と品質のバランスが優秀
claude-opus-4-20250514最高性能。複雑なタスクや長文に最適
claude-haiku-4-20250514最速・最低コスト。簡単な応答に最適
claude-3-5-sonnetClaude 3.5 世代の Sonnet
claude-3-opusClaude 3 世代の最高性能モデル
claude-3-haikuClaude 3 世代の軽量モデル

Gemini (Google) 設定

Google AI の API キーとモデルを設定します。API キーは Google AI Studio から取得できます。

モデル選択:

モデル特徴
gemini-2.0-flash-exp デフォルト最新の高速モデル。マルチモーダル対応
gemini-1.5-pro100万トークンコンテキスト。長文分析に最適
gemini-1.5-flash高速・低コスト版
gemini-2.0-flash-thinking-exp思考プロセス付きの実験的モデル

OpenRouter 設定

OpenRouter は複数の AI プロバイダーのモデルに統一 API でアクセスできるサービスです。API キーは OpenRouter から取得できます。

デフォルトモデル: openrouter/auto - リクエスト内容に応じて最適なモデルを自動選択します。100以上のモデルからドロップダウンで選択することもできます。

ヒント: OpenRouter を使うと、1つの API キーで OpenAI、Claude、Gemini、Llama、Mistral など様々なモデルを試すことができます。

ベクトル埋め込み (RAG)

ナレッジベースやサイト学習データからの検索精度を向上させるベクトル埋め込み検索を有効にします。

設定説明
ベクトル埋め込み有効化有効: AI の埋め込みモデルを使って意味的な検索を行います。キーワード一致ではなく、意味の類似性で検索するため精度が向上します。
無効: 従来の FULLTEXT 検索を使用します。
プロバイダー自動 デフォルト - 選択中の AI プロバイダーの埋め込みモデルを使用
OpenAI - OpenAI の text-embedding モデルを使用
Gemini - Google の embedding モデルを使用
注意: ベクトル埋め込みを有効にすると、チャットメッセージごとに追加の API 呼び出しが発生します。少量の追加コストが発生する場合があります。

MCP (Model Context Protocol) サーバー

MCP サーバーを有効にすると、Claude Desktop、Cursor、その他の MCP 対応ツールからプラグインのナレッジベースにアクセスできるようになります。

設定説明
MCP 有効化有効: REST API エンドポイントを公開し、外部の AI ツールからナレッジベースの検索やサイト情報の取得が可能になります。
無効: MCP エンドポイントを無効にします。
API キー「生成」ボタンで新しい API キーを作成します。このキーは一度だけ表示されます。外部ツールの Bearer トークン認証に使用してください。
エンドポイント URL自動生成される REST API URL。外部ツールの MCP 設定に入力してください。

チャット設定

ボット名

チャットウィンドウのヘッダーとメッセージに表示されるアシスタントの名前です。

デフォルト: Assistant

サイトのブランドに合わせて自由に変更できます。例:「サポート AI」「カスタマーアシスタント」など。

ボットアバター

チャットメッセージの横に表示されるアイコンです。

デフォルト: ロボットの絵文字

絵文字を直接入力するか、「画像をアップロード」ボタンでメディアライブラリから画像を選択できます。画像は自動的にリサイズされます。

ウェルカムメッセージ

チャットボットを開いた時に最初に表示されるメッセージです。

デフォルト: Hello! How can I help you today?

サイトの目的や訪問者に合わせたメッセージに変更することを推奨します。応答言語を「自動検出」に設定している場合は、言語別のウェルカムメッセージを個別に設定できます。

応答言語

AI が応答する言語を制御します。

選択肢説明
サイト言語 デフォルトWordPress の言語設定に従います
自動検出ユーザーの入力言語を検出し、同じ言語で応答します。多言語サイトに最適です。言語別ウェルカムメッセージの設定が追加表示されます。
個別言語 (en, ja, zh, ko, es, fr, de, pt)常に指定した言語で応答します

言語別ウェルカムメッセージ（自動検出モード時のみ表示）:

応答言語を「自動検出」に設定すると、13言語（英語、日本語、中国語、韓国語、スペイン語、フランス語、ドイツ語、ポルトガル語、イタリア語、ロシア語、アラビア語、タイ語、ベトナム語）ごとに個別のウェルカムメッセージを設定できます。ユーザーのブラウザ言語を検出して適切なメッセージを表示します。

システムプロンプト

AI の振る舞い、性格、制約事項を定義する指示文です。すべての会話の裏側で AI に送信されます。

デフォルト: 正確さ、誠実さ、情報の捏造禁止などの基本ルールが設定されています。

ヒント: システムプロンプトには、会社名、サービス内容、回答してはいけないトピック、応答のトーンなどを含めると効果的です。「デフォルトにリセット」ボタンで初期値に戻せます。

コンテキストプロンプト（詳細設定）

ナレッジベースやサイト学習データを AI に渡す際の指示文をカスタマイズできます。通常はデフォルトのまま使用することを推奨します。

プロンプト説明
ナレッジベース完全一致プロンプトナレッジベースに完全一致する質問があった場合の指示。登録済みの回答をそのまま返すよう指示します。
ナレッジベース Q&A プロンプトナレッジベースの Q&A 形式データを参照する際の指示。
サイトコンテキストプロンプトサイト学習（クローラー）で取得したコンテンツを参照する際の指示。

各プロンプトには「デフォルトにリセット」ボタンがあります。

機能プロンプト（詳細設定）

再生成、フィードバック、要約機能の動作をカスタマイズするプロンプトです。

プロンプト説明
再生成プロンプト応答の再生成時に使用される指示文。プレースホルダー: {variation_number}（再生成回数）、{forbidden_start}（前回と異なる開始を指示）、{style}（応答スタイル指定）。
フィードバック良好ヘッダーユーザーが「良い」と評価した応答例を AI に示すヘッダー。
フィードバック不良ヘッダーユーザーが「悪い」と評価した応答例を AI に示すヘッダー。
要約プロンプト会話要約を生成する際の指示文。

メッセージ履歴数

AI に送信する過去のメッセージ数です。この数が多いほど AI は会話の文脈をよく理解しますが、API コストが増加します。

デフォルト: 10 | 範囲: 1 ~ 50

推奨: 一般的なサポートチャットでは 10 ~ 15 が適切です。コスト削減には 5 程度に下げることも可能ですが、AI が前の質問を忘れやすくなります。

最大トークン数

AI の応答1回あたりの最大出力トークン数です。1トークンは日本語で約1文字、英語で約4文字に相当します。

デフォルト: 1000 | 範囲: 100 ~ 4000

GPT-5 自動乗算: GPT-5 モデルでは推論に多くのトークンを消費するため、設定値に自動的に乗算係数（デフォルト: 4倍）が適用されます。例: 1000 と設定すると実際には 4000 トークンが上限になります。

Temperature（創造性）

AI の応答のランダム性を制御します。低い値ほど一貫した応答になり、高い値ほど多様な応答になります。

デフォルト: 0.7 | 範囲: 0 ~ 2（0.1刻み）

値特徴推奨用途
0 ~ 0.3非常に一貫性が高い。同じ質問に同じ回答FAQ、事実ベースの回答
0.5 ~ 0.8バランスの良い応答一般的なカスタマーサポート
0.9 ~ 1.5創造的で多様な応答クリエイティブなコンテンツ
1.5 ~ 2.0非常にランダム。予測困難通常は非推奨

フィードバックボタン

AI の各応答メッセージの下に「いいね」「よくないね」のボタンを表示します。

有効: ユーザーが応答を評価でき、管理者は会話履歴でフィードバックを確認できます。応答品質の改善に役立ちます。

無効: フィードバックボタンを非表示にします。

API 制限超過メッセージ

AI プロバイダーの API クォータを超過した場合にユーザーに表示されるメッセージです。

デフォルト: Currently recharging. Please try again later.

Web 検索

ナレッジベースやサイト学習データで十分な回答が得られない場合に、Web 検索結果を AI の参考情報として利用します。

有効: ナレッジベースに該当する情報がない場合、自動的に Web 検索を実行して回答の参考にします。

無効: ナレッジベースとサイト学習データのみで応答します。

注意: Web 検索を有効にすると、追加の API 呼び出しが発生し、コストが増加する場合があります。

表示設定

テーマ

チャットウィジェットの外観テーマを選択します。Free版では6種類のテーマが利用可能です。

テーマ説明
Default デフォルト標準的なモダンデザイン。丸みを帯びたカード型
Simpleシンプルでミニマルなデザイン
Classicクラシックなチャット UI
Light明るくクリーンなデザイン
Minimal装飾を最小限にしたデザイン
Flatフラットデザイン。影やグラデーションなし

Pro版では追加で10種類のテーマ（Modern、Gradient、Dark、Glass、Rounded、Ocean、Sunset、Forest、Neon、Elegant）とダークモードが利用可能です。PRO

メインカラー

チャットウィジェットのヘッダー、ボタン、バッジなどの主要色です。

デフォルト: #007bff（青）

カラーピッカーで任意の色を選択できます。テーマを選択すると自動的にそのテーマに合った色に変更されます。

バッジ位置

チャットを開くバッジ（フローティングボタン）の画面上の位置です。

選択肢説明
右下 デフォルト最も一般的な配置。ほとんどのサイトに適しています
左下右下に他の要素がある場合に適しています
右上あまり一般的ではありませんが、レイアウトに応じて使用可能
左上特殊なレイアウト用

バッジマージン

バッジの画面端からの距離をピクセル単位で設定します。

右マージン デフォルト: 20px | 下マージン デフォルト: 20px

サイトの他のフローティング要素（Cookie バナーなど）との重なりを避けるために調整してください。

Markdown レンダリング

AI の応答メッセージ内の Markdown 記法をリッチテキストに変換して表示します。

有効 デフォルト: **太字**、*斜体*、リスト、コードブロック、リンクなどが整形されて表示されます。

無効: Markdown 記号がそのまま表示されます。

モバイル表示

モバイルデバイス（スマートフォン、タブレット）でチャットウィジェットを表示するかどうかを制御します。

有効 デフォルト: すべてのデバイスでチャットボットが表示されます。

無効: デスクトップのみでチャットボットが表示されます。モバイルサイトのパフォーマンスを優先する場合に使用します。

ページ表示設定

チャットボットを表示するページを細かく制御できます。

設定デフォルト説明
ホームページ有効サイトのトップページに表示
投稿有効ブログ記事などの投稿ページに表示
固定ページ有効固定ページに表示
アーカイブ有効カテゴリー、タグ、月別アーカイブに表示
特定ページID（含む）空カンマ区切りの投稿/ページ ID。設定すると、このリストのページのみに表示されます（ホワイトリストモード）
特定ページID（除外）空カンマ区切りの投稿/ページ ID。このリストのページでは非表示になります
除外ページ選択空ドロップダウンからページを選択して除外

外部サイト埋め込み

WordPress 以外のサイトにチャットボットを埋め込むためのコードを提供します。

スクリプト埋め込み: フローティングバッジとして表示されます。<script> タグをサイトの </body> 前に挿入してください。

iframe 埋め込み: 固定サイズのチャットウィンドウとして表示されます。任意の場所に <iframe> タグを挿入してください。

reCAPTCHA 設定

reCAPTCHA v3 有効化

Google reCAPTCHA v3 によるボット対策を有効にします。v3 はユーザーに操作を求めず、バックグラウンドでスコアリングします。

有効: チャットメッセージ送信時に reCAPTCHA スコアを検証し、ボットからのメッセージをブロックします。

無効: reCAPTCHA 検証を行いません。

サイトキー / シークレットキー

Google reCAPTCHA 管理コンソールで取得したキーを入力します。

サイトキー: フロントエンドで使用される公開キー。

シークレットキー: サーバーサイドで検証に使用される秘密キー。保存後は暗号化されてマスク表示されます。

スコア閾値

reCAPTCHA v3 のスコア閾値です。このスコア未満のリクエストはブロックされます。

デフォルト: 0.5 | 範囲: 0.0 ~ 1.0

  • 0.1 ~ 0.3: 緩い判定。ほとんどのリクエストを通過させます
  • 0.5: 標準的な判定。推奨値
  • 0.7 ~ 0.9: 厳しい判定。正規ユーザーもブロックされる可能性があります

既存 reCAPTCHA の使用

サイトに既に reCAPTCHA が読み込まれている場合（他のプラグインなど）、二重読み込みを防ぐために有効にします。

失敗時モード

reCAPTCHA の検証自体が失敗した場合（ネットワークエラーなど）の動作を制御します。

選択肢説明
Open（許可） デフォルト検証失敗時もチャットを許可します。可用性を優先する場合に適しています
Closed（ブロック）検証失敗時はチャットをブロックします。セキュリティを最優先する場合に適しています

詳細設定

会話履歴保存

会話の内容をデータベースに保存するかどうかを制御します。

有効 デフォルト: 会話がデータベースに永続化され、管理画面の「会話」ページで確認・分析できます。ユーザーがページを再読み込みした際に会話を復元することもできます。

無効: 会話はセッション中のみ一時的に保持され、ページを離れると失われます。プライバシー重視の場合や、データベースの容量を節約したい場合に使用します。

データ保持期間

会話履歴をデータベースに保持する日数です。期間を過ぎた会話は自動的に削除されます。

デフォルト: 90日 | 0 で無期限保持

アンインストール時データ削除

プラグインをアンインストールした際に、すべてのプラグインデータ（設定、会話履歴、ナレッジベース、インデックスなど）を削除するかどうかを制御します。

有効: アンインストール時にすべてのデータを完全に削除します。

無効 デフォルト: データを保持します。再インストール時にデータを引き継ぎたい場合はこの設定を使用します。

WP Consent API 厳格モード

WP Consent API プラグインと連携して、GDPR 対応のクッキー同意を要求します。

有効: WP Consent API プラグインが有効で、ユーザーが同意していない場合、localStorage を使用しません。セッション ID やチャット履歴がブラウザに保存されなくなります。

無効 デフォルト: 同意確認なしに localStorage を使用します。

レート制限

ユーザーあたりのメッセージ送信頻度を制限して、API コストの急増やスパムを防止します。

設定デフォルト説明
最大リクエスト数20指定時間内に許可する最大メッセージ数。0 で無制限。
時間ウィンドウ1時間レート制限の時間枠。選択肢: 1分、5分、10分、30分、1時間、3時間、6時間、12時間、1日
ヒント: Pro版では分単位と時間単位の2段階レート制限（バースト制限 + 持続制限）が利用可能です。

プロキシ設定

Cloudflare やロードバランサーを使用している場合の IP アドレス検出を設定します。

設定説明
Cloudflare IP を信頼有効: CF-Connecting-IP ヘッダーから実際のクライアント IP を取得します。Cloudflare を使用している場合に有効にしてください。
プロキシ IP を信頼有効: X-Forwarded-For ヘッダーから実際のクライアント IP を取得します。リバースプロキシを使用している場合に有効にしてください。
注意: これらの設定を不適切に有効にすると、IP スプーフィングのリスクがあります。使用しているインフラに合わせて設定してください。

サイト学習（クローラー）

クローラー有効化

WordPress サイトのコンテンツを自動的にクロールし、AI の参考知識としてインデックスに登録します。

有効 デフォルト: 指定した投稿タイプのコンテンツを定期的にクロールし、チャンクに分割してインデックスに保存します。ユーザーの質問に関連するコンテンツが自動的に AI の回答に反映されます。

無効: クロールを実行しません。ナレッジベースの手動登録のみで運用する場合に使用します。

投稿タイプ

クロール対象の WordPress 投稿タイプを選択します。

デフォルト: すべて

「すべて」を選択するとサイト内の全投稿タイプ（投稿、固定ページ、カスタム投稿タイプ）がクロールされます。特定の投稿タイプのみを選択することもできます。

クロール間隔

自動クロールの実行間隔です。

選択肢説明
毎日 デフォルト1日1回クロールを実行。コンテンツの更新が頻繁なサイトに適しています
毎週週1回クロールを実行。コンテンツの更新が少ないサイトに適しています

手動でクロールを実行することもできます。ダッシュボードの「サイト学習」セクションから「今すぐクロール」ボタンをクリックしてください。

チャンクサイズ

コンテンツを分割するチャンク（断片）1つあたりの文字数です。

デフォルト: 1000文字

  • 小さい値（500 ~ 800）: より精密な検索が可能ですが、インデックスサイズが増加します
  • 大きい値（1500 ~ 2000）: より広い文脈を保持しますが、検索の精度が低下する場合があります

最大結果数

ユーザーの質問に対して AI に渡すコンテキスト（参考情報）チャンクの最大数です。

デフォルト: 3

  • 少ない値（1 ~ 2）: API コストを節約しますが、関連情報が不足する場合があります
  • 多い値（5 ~ 10）: より多くの参考情報を AI に渡しますが、API コストが増加します

除外 ID

クロールから除外する投稿/ページの ID を指定します。プライバシーポリシーや利用規約など、チャットの参考にしたくないコンテンツを除外する際に使用します。

ナレッジベース

概要

ナレッジベースは、AI が回答に使用する Q&A データのコレクションです。サイト学習（クローラー）とは異なり、手動で登録した正確な回答を優先的に使用します。

Free版の制限: ナレッジベースには最大20件まで登録できます。Pro版では無制限に登録可能です。

Q&A の追加

管理画面の「ナレッジ」ページから、質問と回答のペアを手動で追加できます。

  • 質問: ユーザーが問い合わせる可能性のある質問文
  • 回答: AI が返すべき正確な回答文
  • カテゴリー: 整理用のラベル（オプション）
ヒント: よくある質問を登録しておくと、AI が正確な回答を返せるようになります。質問は複数パターン登録すると認識精度が向上します。

CSV インポート

CSV ファイルからナレッジベースを一括インポートできます。CSV フォーマット:

質問,回答,カテゴリー
"料金はいくらですか？","月額1000円からご利用いただけます。","料金"
"営業時間は？","平日9時から18時まで営業しています。","営業"

会話履歴

会話一覧

管理画面の「会話」ページで、すべてのチャット会話を一覧表示できます。各会話のセッション ID、開始日時、メッセージ数、ステータスが表示されます。

メッセージ表示

会話をクリックすると、ユーザーと AI のメッセージのやり取りを時系列で確認できます。フィードバック（いいね/よくないね）の評価も表示されます。

CSV/JSON エクスポート機能は Pro版で利用可能です。PRO
