Rapls AI Chatbot Manual (Pro Edition)

The Pro edition includes all Free features plus the advanced capabilities documented below.
For settings available in the Free version, please refer to the Free Manual.

Customer

Lead Capture

Collect visitor information before or during a chat session. Lead data is stored in the database and can be exported as CSV or JSON. Optional email notification sends captured leads to your team in real time.

Lead Capture Enabled

Master toggle to activate the lead capture form. When enabled, a form is displayed to the visitor based on the "Lead Form Required" setting below.

OptionDescription
EnabledShow the lead form to visitors
DisabledNo lead form is shown; visitors chat immediately

Default: Disabled

Lead Form Required

Controls whether the visitor must complete the lead form before chatting.

OptionDescription
RequiredThe visitor cannot send a message until the form is submitted
OptionalThe form is shown but the visitor may skip it and begin chatting

Default: Required

Form Title

The heading displayed at the top of the lead capture form. Keep it concise and welcoming.

Default: Before we start

Form Description

Supporting text shown below the form title. Use this to explain why you are asking for the visitor's information and how it will be used.

Default: Please enter your information

Standard Fields:

Four built-in fields are available. Each field can be independently enabled or disabled, and marked as required or optional.

FieldDefault EnabledDefault RequiredNotes
NameYesYesVisitor's full name
EmailYesYesValidated as a proper email address
PhoneYesNoFree-form telephone input
CompanyYesNoCompany or organization name

Custom Fields:

Add unlimited custom fields to the lead form. Each field supports the following properties:

PropertyDescription
LabelThe field label displayed to the visitor
TypeInput type: text, email, tel, textarea, or select
RequiredWhether the field must be filled before submission
OptionsFor select fields only: comma-separated list of choices
Tip: Use the select type for fields like "Department" or "Inquiry Type" to keep responses consistent and make CSV exports easier to analyze.

Email Notification:

Sends an email to your team whenever a new lead is captured. The email contains all submitted field values.

SettingDescription
EnabledTurn email notifications on or off
Recipient EmailThe email address that receives notifications. Uses the site admin email if left blank.

Offline Messages

When business hours are configured and the current time falls outside those hours, visitors see an offline form instead of the chat widget. This lets you collect messages while your team is away.

SettingDefaultDescription
Offline Message EnabledDisabledMaster toggle. When enabled and outside business hours, the offline form replaces the chat interface.
Form TitleWe are currently offlineThe heading shown on the offline message form
Form Description(empty)Supporting text below the title. Use this to set expectations for when the visitor will receive a response.

Email Notification:

Sends an email when a visitor submits an offline message.

SettingDescription
EnabledTurn email notifications on or off for offline messages
Recipient EmailThe address that receives offline messages. Uses the site admin email if left blank.
Tip: Pair Offline Messages with the Business Hours feature. When business hours are not enabled, the offline form will not appear automatically. You can use offline messages as a standalone feature by enabling it even without business hours -- visitors will see the offline form whenever AI is unavailable.

Conversion Tracking

Track when chat conversations lead to desired outcomes by monitoring page visits after a chat session. When a visitor navigates to a URL matching one of your goal patterns, the conversion is recorded and linked to their conversation.

Conversion Tracking Enabled

Master toggle to activate conversion tracking for all chat sessions.

Default: Disabled

Goal Definitions:

Define one or more conversion goals. Each goal consists of three properties:

PropertyDescription
URL PatternA string to match against page URLs. Supports partial matching (e.g., /thank-you matches any URL containing that path). You can use * as a wildcard.
Goal NameA descriptive label for this conversion (e.g., "Purchase Complete", "Contact Form Sent")
Goal ValueAn optional numeric value representing the monetary worth of this conversion (e.g., 50 for a $50 purchase)
Tip: Conversion data appears in the Analytics dashboard. Use Goal Value to calculate ROI and measure the chatbot's business impact.

AI Enhancement

Related Suggestions

After each AI response, display a set of related follow-up questions the visitor can click. The AI generates these suggestions based on the current conversation context.

Default: Disabled

Suggestions Prompt: Customize the prompt used to generate related follow-up questions. Control the style, number, and relevance of suggestions.

Tip: Related suggestions increase engagement by guiding visitors to deeper topics. They work especially well on FAQ-heavy sites.

Autocomplete

As the visitor types, the chatbot suggests completions based on common queries and the knowledge base. The visitor can accept a suggestion by pressing Tab or clicking it.

Default: Disabled

Prompt Templates

Load a pre-built system prompt optimized for a specific industry or use case. Templates provide the AI with domain-specific instructions, tone, and vocabulary.

TemplateUse Case
Customer SupportGeneral helpdesk with escalation awareness and empathetic tone
E-commerceProduct inquiries, order status, returns, and recommendations
EducationCourse information, enrollment, and student guidance
FAQConcise answers drawn from a knowledge base
AppointmentsScheduling, availability checks, and booking confirmations

Tip: Selecting a template pre-fills the system prompt. You can further customize the text after loading a template. The template provides a starting point -- your edits take precedence.

Regenerate

Shows a "Regenerate" button below each AI response, allowing the visitor to request a new answer to the same question. Useful when the first response is unsatisfactory.

Default: Enabled

The regeneration prompt can be customized to control the style and constraints of regenerated responses.

Sentiment Analysis

Detect the visitor's emotional state during a conversation and adjust the AI's tone in real time. The AI analyzes each message for sentiment and modifies its response style accordingly.

Default: Disabled

SentimentAI Response
FrustratedMore patient tone, acknowledges frustration, offers clear next steps
ConfusedSimplified language, step-by-step explanations, clarifying questions
UrgentPrioritizes speed, provides direct answers, escalates if necessary
PositiveMatches upbeat tone, reinforces satisfaction, suggests additional resources
NegativeEmpathetic response, avoids defensive language, offers resolution paths

Sentiment Detection Prompt: Customize the prompt used to classify the visitor's sentiment. You can adjust this to recognize industry-specific emotional cues.

Tone Adjustments: Define how the AI adjusts its response for each detected sentiment. Each adjustment is a prompt fragment appended to the system prompt for that turn.

Note: Sentiment analysis adds an extra AI call per message. If response speed is critical, consider disabling it and using a well-crafted system prompt to handle tone instead.

Context Memory

Remember key information about returning visitors across separate chat sessions. Context memory extracts facts like the visitor's name, preferences, and past issues, then injects them into future conversations for a personalized experience.

SettingDefaultDescription
Context Memory EnabledDisabledStores extracted context in the wp_aichat_user_context database table
Retention Days30Number of days to retain stored context. Old entries are automatically purged.

Context Extraction Prompt: The prompt used at the end of each conversation to extract memorable facts (name, preferences, issues mentioned, etc.).

Context Injection Prompt: The prompt template used to inject stored context into new conversations. Placeholders are replaced with the visitor's stored context before being prepended to the system prompt.

Tip: Context memory is especially valuable for membership sites or SaaS products where users return frequently. Combine with lead capture to link context to a known identity.

Quick Replies

Display clickable quick-reply buttons below the chat input. Visitors can tap a button to instantly send a predefined message without typing. Useful for common questions or menu-style navigation.

Default: Disabled

Each reply is defined with a Text (button label) and an Action (the message sent to the AI when clicked; if blank, the button text is sent as-is).

Tip: Use quick replies as a navigation menu: "Pricing", "Support Hours", "Book a Demo". The action can be a longer, more specific prompt that produces better AI responses than the short button text alone.

Multimodal (Image Upload)

Allow visitors to upload images alongside their messages. The AI can analyze the image content and respond accordingly.

SettingDefaultDescription
Image Upload EnabledDisabledShows an image upload button in the chat input area
Max Size2048 KBMaximum file size for uploaded images
Allowed Formatsjpg, png, gif, webpFile formats accepted for upload
Requirement: Requires an AI provider that supports vision capabilities (GPT-4o, Claude, Gemini). Image analysis consumes more tokens than text-only messages.

Response Cache

Cache AI responses to reduce API costs and improve response times. When a visitor asks a question that matches a previously cached response (based on SHA-256 hashing), the cached response is served instantly without calling the AI provider.

SettingDefaultDescription
Cache EnabledDisabledMaster toggle for response caching
TTL (Time to Live)7 daysNumber of days a cached response remains valid

The admin panel displays cache statistics (hit rate, total cached responses, storage size) and provides a button to manually clear all cached responses.

Tip: Response caching is most effective on FAQ-heavy sites where many visitors ask similar questions. Sites with highly personalized conversations may see lower hit rates.

WooCommerce Integration

Automatically index WooCommerce product data so the AI can answer product-related questions and display rich product cards in chat responses.

Product Cards: Default: Enabled - When the AI discusses specific products, visual cards (image, title, price, link) are displayed directly in chat responses.

Auto-Crawl Product Data: Automatically includes WooCommerce product data (title, description, price, categories, attributes) in the site learning crawler. Products are indexed alongside your regular page content.

Tip: Make sure to run a site crawl after enabling WooCommerce integration to index your existing products. New products are picked up on subsequent scheduled crawls.

Operations

Business Hours

Define when your chatbot operates in "online" mode. Outside business hours, you can show an offline message, disable the chatbot, or continue with AI-only responses. Business hours also feed into the {business_hours} dynamic variable.

SettingDefaultDescription
Business Hours EnabledDisabledWhen disabled, the chatbot operates 24/7 with no time-based restrictions
TimezoneAsia/TokyoThe timezone used to evaluate business hours
Outside Hours Message--The message displayed to visitors who open the chatbot outside business hours

Per-Day Settings: Configure each day of the week independently with enabled/disabled, start time, and end time (24-hour format). Default: Monday-Friday 9:00-18:00, Saturday/Sunday disabled.

Holidays: Override business hours on specific dates. Add dates in YYYY-MM-DD format, one per line. A special holiday message can be configured.

Keywords (EN)--English trigger phrases (e.g., "talk to a human", "speak to agent", "real person")

Notification Methodemailemail: Email notification

AI Forms

Create intelligent forms that use AI to process submissions. Unlike standard contact forms, AI Forms send the collected data to the AI along with a custom prompt, generating a personalized response for the visitor.

SettingDescription
Form NameDisplay name for admin reference
DescriptionText shown above the form fields
System PromptThe system prompt used when processing this form's submissions
PromptTemplate with {field_key} placeholders to inject field values
FieldsEach field: label, key, type (text, email, tel, textarea, select), required, options

Email Notification: Optionally send an email when a form is submitted, including all field values and the AI's response.

Shortcode: [wpaic_form id="X"] -- embed on any page or post.

Tip: AI Forms are ideal for quote calculators, diagnostic tools, personalized recommendations, and interactive assessments. The AI processes all field data together to generate a contextual response.

Actions (Intent Recognition)

Define automatic actions triggered when the AI recognizes a visitor's intent. Actions can open URLs, trigger forms, start scenarios, or initiate handoffs based on what the visitor is trying to accomplish.

Each action consists of an Intent (keyword/phrase), an Action Type (open URL, trigger form, start scenario, initiate handoff), and a Target (URL, form ID, scenario ID, etc.).

Scenarios (Conversation Flows)

Build multi-step guided conversation flows with branching logic, conditions, and automated actions at each step. Useful for troubleshooting wizards, qualification flows, and onboarding sequences.

PropertyDescription
IDUnique identifier for the scenario
NameDisplay name shown in admin and optionally to visitors
DescriptionInternal description for admin reference
Trigger TypeHow the scenario is activated: keyword match, intent detection, manual trigger, or page URL
StepsAn ordered list of steps, each with a message, input options, conditions, and a next-step mapping
Tip: Start with a simple 3-5 step flow to test the feature. Each step can include quick-reply buttons that guide the visitor to the next step, creating a decision-tree experience.

Multi-Bot

Deploy multiple chatbot configurations on different pages or for different purposes. Each bot has its own system prompt and model setting, allowing specialized behavior per context (e.g., a sales bot on the pricing page and a support bot on the help center).

PropertyDescription
IDUnique identifier used for page assignment
NameDisplay name for admin reference
System PromptThe system prompt specific to this bot
ModelThe AI model used by this bot (can differ from the global default)
Tip: Multi-bot works with intent routing. The system can automatically direct conversations to the most appropriate bot based on the visitor's initial question.

Voice Input / Text-to-Speech

Voice Input (STT): Visitors can speak their questions using the browser's speech recognition API. A microphone button appears in the chat input area. Uses the Web Speech API built into modern browsers.

Text-to-Speech (TTS): A speaker button appears on each AI response. Clicking it reads the response text aloud using the browser's built-in speech synthesis engine.

Note: Voice features depend on browser support. The Web Speech API is available in Chrome, Edge, and Safari. Firefox has partial support. Mobile browsers generally support voice input but TTS availability varies.

Screen Sharing

Allow visitors to capture and share screenshots of their screen within the chat. The screenshot is sent as an image attachment that the AI or operator can analyze. Uses the browser's native screen capture API.

Default: Disabled

Note: Screen capture requires browser permission from the visitor. Not all browsers support the Screen Capture API. The feature degrades gracefully on unsupported browsers.

Analytics & Management

Analytics Dashboard

A comprehensive analytics dashboard built with Chart.js, accessible from the Pro admin panel. Provides visual insights into chatbot performance and visitor behavior.

  • Overview: Total visitors, conversations, messages, satisfaction score, conversion rate
  • Time Analysis: Usage by hour of day and day of week to identify peak times
  • Device Analysis: Desktop/mobile/tablet usage breakdown
  • Page Analysis: Pages generating the most chatbot interactions
  • FAQ Ranking: Most frequently asked questions
  • Unresolved Questions: Questions the AI could not answer satisfactorily
  • Knowledge Gap Detection: Automatic detection of missing information in the knowledge base
  • AI Quality Score: Aggregate score based on feedback ratings, resolution rates, and conversation length
  • Churn/Bounce Analysis: Identifies conversations where visitors left without resolution
  • Real-Time Monitor: Live view of currently active conversations with operator intervention capability

PDF/Print Report: Generate a printable PDF report using server-side rendering (Dompdf). Includes all charts and metrics formatted for presentation. Japanese font (Noto Sans JP) supported.

Backup & Maintenance

  • Settings Export: Download all plugin settings as a JSON file. API keys are excluded for security.
  • Settings Import: Upload a previously exported JSON file to restore settings. A confirmation dialog prevents accidental overwrites.
  • Knowledge Export: Export the knowledge base as CSV or JSON for backups, migration, or external analysis.
Tip: Export settings before making major changes. You can restore them by importing the JSON file.

Pro Themes & Dark Mode

Test Mode

Make the chatbot visible only to administrators. Use this to test changes to system prompts, knowledge base entries, or feature configurations before going live.

Multisite Support

Full WordPress multisite compatibility. Network administrators can configure global default settings that apply to all sites, while individual site administrators can override specific settings. A "Reset to Network Default" button is available for each setting.

FAQ Auto-Generation

Generate draft knowledge base entries from detected knowledge gaps. The AI creates question-and-answer pairs based on unresolved visitor questions. Drafts go through a review workflow before being published to the knowledge base.

AI Admin Tools

  • AI Content Editor Sidebar: A sidebar panel in the WordPress post editor that uses AI to suggest content improvements based on your chatbot's knowledge base.
  • Response Edit Suggestions: AI suggests improved responses for messages that received negative feedback. Accept a suggestion to add it to the knowledge base.
  • Similar Question Merge: Detects knowledge base entries with similar questions and suggests merging them to reduce redundancy.

Chat Features

  • File Upload: Allow visitors to upload files (documents, PDFs, images) alongside messages.
  • Chat Bookmarks: Visitors can bookmark specific messages for easy reference.
  • Conversation Search: Full-text search across all messages within a conversation.
  • Conversation Sharing: Generate shareable read-only links for conversations.

Performance

  • Similar Question Cache: Extends response cache by matching semantically similar questions using word-level similarity scoring.
  • Batch Processing: Process multiple knowledge base operations in batches to prevent timeout errors.
  • Performance Monitoring: Track API response times, queue lengths, and error rates.
  • Queue Management: Control concurrent API requests to prevent provider rate limiting.

Testing & Staging

  • Custom Metadata Fields: Attach custom key-value metadata to conversations via REST API or dynamic variables.
  • Staging Mode: Creates a separate settings environment. Changes do not affect live chatbot until published.
  • Approval Workflow: Require administrator approval before knowledge base or setting changes go live.
