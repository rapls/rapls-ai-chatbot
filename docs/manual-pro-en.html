Lead Capture

Collect visitor information before or during a chat session. Lead data is stored in the database and can be exported as CSV or JSON. Optional email notification sends captured leads to your team in real time.

Lead Capture Enabled

Master toggle to activate the lead capture form. When enabled, a form is displayed to the visitor based on the "Lead Form Required" setting below.

OptionDescription
EnabledShow the lead form to visitors
DisabledNo lead form is shown; visitors chat immediately

Default: Disabled

Lead Form Required

Controls whether the visitor must complete the lead form before chatting.

OptionDescription
RequiredThe visitor cannot send a message until the form is submitted
OptionalThe form is shown but the visitor may skip it and begin chatting

Default: Required

Form Title

The heading displayed at the top of the lead capture form. Keep it concise and welcoming.

Default: Before we start

Form Description

Supporting text shown below the form title. Use this to explain why you are asking for the visitor's information and how it will be used.

Default: Please enter your information

Standard Fields

Four built-in fields are available. Each field can be independently enabled or disabled, and marked as required or optional.

FieldDefault EnabledDefault RequiredNotes
NameYesYesVisitor's full name
EmailYesYesValidated as a proper email address
PhoneYesNoFree-form telephone input
CompanyYesNoCompany or organization name

Custom Fields

Add unlimited custom fields to the lead form. Each field supports the following properties:

PropertyDescription
LabelThe field label displayed to the visitor
TypeInput type: text, email, tel, textarea, or select
RequiredWhether the field must be filled before submission
OptionsFor select fields only: comma-separated list of choices
Tip: Use the select type for fields like "Department" or "Inquiry Type" to keep responses consistent and make CSV exports easier to analyze.

Email Notification

Sends an email to your team whenever a new lead is captured. The email contains all submitted field values.

SettingDescription
EnabledTurn email notifications on or off
Recipient EmailThe email address that receives notifications. Uses the site admin email if left blank.

Offline Messages

When business hours are configured and the current time falls outside those hours, visitors see an offline form instead of the chat widget. This lets you collect messages while your team is away.

Offline Message Enabled

Master toggle. When enabled and outside business hours, the offline form replaces the chat interface.

Default: Disabled

Form Title

The heading shown on the offline message form.

Default: We are currently offline

Form Description

Supporting text below the title. Use this to set expectations for when the visitor will receive a response.

Default: (empty)

Email Notification

Sends an email when a visitor submits an offline message.

SettingDescription
EnabledTurn email notifications on or off for offline messages
Recipient EmailThe address that receives offline messages. Uses the site admin email if left blank.
Tip: Pair Offline Messages with the Business Hours feature. When business hours are not enabled, the offline form will not appear automatically. You can use offline messages as a standalone feature by enabling it even without business hours -- visitors will see the offline form whenever AI is unavailable.

Conversion Tracking

Track when chat conversations lead to desired outcomes by monitoring page visits after a chat session. When a visitor navigates to a URL matching one of your goal patterns, the conversion is recorded and linked to their conversation.

Conversion Tracking Enabled

Master toggle to activate conversion tracking for all chat sessions.

Default: Disabled

Goal Definitions

Define one or more conversion goals. Each goal consists of three properties:

PropertyDescription
URL PatternA string to match against page URLs. Supports partial matching (e.g., /thank-you matches any URL containing that path). You can use * as a wildcard.
Goal NameA descriptive label for this conversion (e.g., "Purchase Complete", "Contact Form Sent")
Goal ValueAn optional numeric value representing the monetary worth of this conversion (e.g., 50 for a $50 purchase)
Tip: Conversion data appears in the Analytics dashboard. Use Goal Value to calculate ROI and measure the chatbot's business impact.

Improve the quality and interactivity of AI responses with suggestions, autocomplete, prompt templates, and response regeneration.

Sentiment Analysis

Detect the visitor's emotional state during a conversation and adjust the AI's tone in real time. The AI analyzes each message for sentiment and modifies its response style accordingly.

Sentiment Analysis Enabled

Master toggle. When enabled, every visitor message is evaluated for emotional tone before generating a response.

Default: Disabled

Sentiment Detection Prompt

The prompt used internally to classify the visitor's sentiment. You can customize this to recognize industry-specific emotional cues. The prompt should instruct the AI to return one of the recognized sentiment categories.

Tone Adjustments

Define how the AI adjusts its response when each sentiment is detected. Each adjustment is a prompt fragment appended to the system prompt for that turn.

SentimentPurpose
FrustratedPrompt the AI to be more patient, acknowledge the frustration, and offer clear next steps
ConfusedPrompt the AI to simplify language, provide step-by-step explanations, and ask clarifying questions
UrgentPrompt the AI to prioritize speed, provide direct answers, and escalate if necessary
PositivePrompt the AI to match the upbeat tone, reinforce satisfaction, and suggest additional resources
NegativePrompt the AI to be empathetic, avoid defensive language, and offer resolution paths
Tip: Sentiment analysis adds an extra AI call per message. If response speed is critical, consider disabling it and using a well-crafted system prompt to handle tone instead.

Context Memory

Remember key information about returning visitors across separate chat sessions. Context memory extracts facts like the visitor's name, preferences, and past issues, then injects them into future conversations for a personalized experience.

Context Memory Enabled

Master toggle. When enabled, the plugin stores extracted context in the wp_aichat_user_context database table.

Default: Disabled

Retention Days

Number of days to retain stored context. After this period, old context entries are automatically purged by the cleanup cron job.

Default: 30

Context Extraction Prompt

The prompt used at the end of each conversation to extract memorable facts. The AI reads the conversation history and returns structured context (name, preferences, issues mentioned, etc.).

Context Injection Prompt

The prompt template used to inject stored context into new conversations. Placeholders in this template are replaced with the visitor's stored context before being prepended to the system prompt.

Tip: Context memory is especially valuable for membership sites or SaaS products where users return frequently. Combine with lead capture to link context to a known identity.

Quick Replies

Display clickable quick-reply buttons below the chat input. Visitors can tap a button to instantly send a predefined message without typing. Useful for common questions or menu-style navigation.

Quick Replies Enabled

Master toggle to show or hide quick-reply buttons.

Default: Disabled

Reply Definitions

Define each quick-reply button with the following properties:

PropertyDescription
TextThe label displayed on the button
ActionThe message sent to the AI when the button is clicked. If blank, the button text is sent as-is.
Tip: Use quick replies as a navigation menu: "Pricing", "Support Hours", "Book a Demo". The action can be a longer, more specific prompt that produces better AI responses than the short button text alone.

Multimodal (Image Upload)

Allow visitors to upload images alongside their messages. The AI can analyze the image content and respond accordingly. Requires an AI provider that supports vision capabilities (OpenAI GPT-4o, Claude, Gemini).

Image Upload Enabled

Master toggle. Shows an image upload button in the chat input area.

Default: Disabled

Max Size

Maximum file size for uploaded images in kilobytes.

Default: 2048 KB (2 MB)

Allowed Formats

File formats accepted for upload.

Default: jpg, png, gif, webp

Note: Image analysis consumes more tokens than text-only messages. Monitor your API usage when enabling this feature.

Response Cache

Cache AI responses to reduce API costs and improve response times. When a visitor asks a question that matches a previously cached response (based on SHA-256 hashing), the cached response is served instantly without calling the AI provider.

Cache Enabled

Master toggle for response caching.

Default: Disabled

TTL (Time to Live)

Number of days a cached response remains valid. After expiration, the next identical question triggers a fresh API call.

Default: 7 days

Cache Statistics

The admin panel displays cache hit rate, total cached responses, and storage size. Use this to evaluate the effectiveness of caching on your site.

Clear Cache

A button in the admin panel to manually purge all cached responses. Use this after making significant changes to the system prompt or knowledge base.

Tip: Response caching is most effective on FAQ-heavy sites where many visitors ask similar questions. Sites with highly personalized conversations may see lower hit rates.

WooCommerce Integration

Automatically index WooCommerce product data so the AI can answer product-related questions and display rich product cards in chat responses.

Product Cards Enabled

When enabled, the AI displays product information as visual cards (image, title, price, link) directly in chat responses when discussing specific products.

Default: Enabled

Auto-Crawl Product Data

Automatically includes WooCommerce product data (title, description, price, categories, attributes) in the site learning crawler. Products are indexed alongside your regular page content.

Tip: Make sure to run a site crawl after enabling WooCommerce integration to index your existing products. New products are picked up on subsequent scheduled crawls.

Handoff (Human Escalation)

Transfer a conversation from the AI to a human operator. Handoff can be triggered automatically by keyword detection or manually by the visitor. When triggered, the system sends a notification and the operator can take over the conversation in real time.

Handoff Enabled

Master toggle for the human handoff feature.

Default: Disabled

Auto-Detect

Automatically trigger handoff when the visitor uses specific keywords indicating they want to speak to a human.

SettingDescription
EnabledToggle auto-detection on or off
Keywords (EN)English trigger phrases (e.g., "talk to a human", "speak to agent", "real person")
Keywords (JA)Japanese trigger phrases (e.g., "人間", "オペレーター", "担当者")

Default: Enabled

Notification Email

The email address notified when a handoff is triggered. Uses the site admin email if left blank.

Handoff Message

The message shown to the visitor when the handoff is initiated (e.g., "Connecting you to a team member. Please wait a moment.").

Notification Method

How the operator is notified of a new handoff request.

OptionDescription
emailSend an email notification only
slackSend a Slack notification only (requires Slack webhook URL)
bothSend both email and Slack notifications

Slack Webhook URL

The Slack incoming webhook URL used for handoff notifications. Required when the notification method includes Slack.

Auto-Close

Automatically close the handoff request if no operator responds within the specified time. The visitor is informed that the handoff timed out and is returned to AI mode.

Default: 30 minutes (Range: 5-1440 minutes)

Operator Mode

When an operator takes over a conversation, the AI is paused and the operator can type responses directly. The operator sees the full conversation history and any context memory for the visitor.

SettingDescription
EnabledToggle operator mode availability
Takeover MessageThe message shown to the visitor when an operator takes over (e.g., "A team member has joined the conversation.")

Webhook

Send real-time event notifications to an external URL. Webhooks let you connect the chatbot to CRMs, automation platforms (Zapier, Make, n8n), and custom backends.

Webhook Enabled

Master toggle for webhook delivery.

Default: Disabled

Webhook URL

The endpoint that receives POST requests with event data in JSON format.

HMAC Signature Secret

An optional shared secret used to sign webhook payloads. When set, each request includes an X-Signature header containing an HMAC-SHA256 hash of the request body. Use this to verify that incoming webhooks are authentic.

Events

Select which events trigger a webhook delivery:

EventDescription
new_conversationFired when a new chat session is created
new_messageFired when either the visitor or AI sends a message
lead_capturedFired when a visitor submits the lead form
handoff_requestedFired when a handoff to a human is triggered

Test Button

Click to send a test payload to your webhook URL. The test payload includes sample data for all selected events. Check your endpoint for a 200 response to confirm the connection works.

Tip: Use a service like webhook.site to inspect payloads during development before connecting your production endpoint.

Voice Input (STT) & Text-to-Speech (TTS)

Add voice capabilities to the chatbot. Visitors can speak their questions using the browser's speech recognition API, and the AI's responses can be read aloud using text-to-speech.

Voice Input (Speech-to-Text)

When enabled, a microphone button appears in the chat input area. Visitors click the button and speak; their speech is converted to text and entered into the message field. Uses the Web Speech API built into modern browsers.

Text-to-Speech

When enabled, a speaker button appears on each AI response. Clicking it reads the response text aloud using the browser's built-in speech synthesis engine. The voice, rate, and language are determined by the browser's settings.

Note: Voice features depend on browser support. The Web Speech API is available in Chrome, Edge, and Safari. Firefox has partial support. Mobile browsers generally support voice input but TTS availability varies.

AI Admin Tools

AI Content Editor Sidebar

A sidebar panel available in the WordPress post editor that uses AI to suggest improvements to your content. The AI analyzes your draft and offers suggestions for clarity, SEO, tone, and completeness based on your chatbot's knowledge base.

Response Edit Suggestions

When reviewing conversations in the admin panel, the AI can suggest improved responses for messages that received negative feedback. Accept a suggestion to add it to the knowledge base, improving future responses to similar questions.

FAQ Auto-Generation

Generate draft knowledge base entries from detected knowledge gaps. The AI creates question-and-answer pairs based on unresolved visitor questions. Drafts go through a review workflow before being published to the knowledge base.

Similar Question Merge

Detects knowledge base entries with similar questions (using word similarity scoring) and suggests merging them. Reduces redundancy and improves the AI's ability to find relevant answers.

Chat Features

File Upload

Allow visitors to upload files (documents, PDFs, images) alongside their messages. Uploaded files are stored in the WordPress media library and linked to the conversation.

Chat Bookmarks

Visitors can bookmark specific messages in a conversation for easy reference. Bookmarked messages are highlighted and accessible via a bookmarks panel.

Conversation Search

Full-text search across all messages within a conversation. Available both to visitors in the chat widget and to admins in the conversation viewer.

Conversation Sharing

Generate a shareable link for a conversation. The recipient can view the full conversation history in a read-only view without logging in. Useful for sharing support interactions with team members.

Performance

Similar Question Cache

Extends the response cache by matching semantically similar questions (not just exact matches). Uses word-level similarity scoring to find cached responses for questions phrased differently but with the same meaning.

Batch Processing

Process multiple knowledge base operations (import, update, export) in batches to prevent timeout errors on large datasets.

Performance Monitoring

Track API response times, queue lengths, and error rates. The monitoring dashboard shows historical performance data so you can identify degradation early.

Queue Management

Control the number of concurrent API requests. When the chatbot receives more simultaneous requests than the limit, additional requests are queued and processed in order. This prevents API rate limiting from your AI provider.

Testing & Staging

Test Mode

Enable test mode to make the chatbot visible only to administrators. Visitors see no chatbot. Use this to test changes to system prompts, knowledge base entries, or feature configurations before going live.

Custom Metadata Fields

Attach custom key-value metadata to conversations. Metadata can be set via the REST API or through dynamic variables. Use it to tag conversations with campaign IDs, referral sources, or A/B test variants.

Staging Mode

Creates a separate settings environment for testing. Changes made in staging mode do not affect the live chatbot until you publish them.

Approval Workflow

Require administrator approval before knowledge base changes or setting modifications go live. Useful for teams where multiple people manage the chatbot. Changes are queued for review and can be approved or rejected.

Multisite Support

When network-activated, a "Network Settings" page appears in the network admin dashboard. Settings configured here serve as defaults for all sites in the network. Individual sites can override any setting.

Per-Site Overrides

Site administrators see all standard settings and can customize the chatbot for their specific site. A "Reset to Network Default" button is available for each setting to revert to the network-wide value.

Audit Logs

Track all administrative actions performed on the chatbot. Every setting change, knowledge base edit, export, and configuration modification is recorded with a timestamp and the acting user.

Viewing Logs

Access the audit log from the Pro admin panel. Logs are displayed in reverse chronological order with filters for date range, user, and action type.

CSV Export

Export audit log entries as a CSV file for external analysis or compliance record-keeping.

Retention Policy

Configure how long audit log entries are retained. Older entries are automatically purged. Set this based on your compliance requirements.
