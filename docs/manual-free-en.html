Rapls AI Chatbot Manual (Free Edition)

Complete documentation for every setting in the WordPress AI Chatbot plugin.
Version: 1.5.0 | PHP 7.4+ | WordPress 5.8+

AI Settings

AI Provider

Select the AI service your chatbot will use. Each provider requires its own API key and model selection.

OptionDescription
OpenAI DefaultGPT-4o, GPT-4 Turbo, GPT-3.5 Turbo, and more. The most widely used AI provider.
Claude (Anthropic)Claude Opus, Sonnet, Haiku models. Excels at long-form understanding and safety.
Gemini (Google)Gemini 2.0 Flash, 1.5 Pro, and more. Strong multimodal capabilities.
OpenRouterGateway service providing unified API access to 100+ models from multiple providers.
Tip: Switching providers preserves your conversation history, but response style may vary between different models.

Configure your OpenAI API key and model. Get your API key from the OpenAI Dashboard.

API Key: Enter a key starting with sk-. It will be encrypted on save. Use the "Test Connection" button to verify.

OpenRouter Settings

OpenRouter provides unified API access to 100+ models from multiple providers. Get your key from OpenRouter.

Default Model: openrouter/auto - Automatically selects the best model for each request. You can also choose specific models from the dropdown.

Vector Embedding (RAG)

Enables semantic search using AI embeddings to improve knowledge base retrieval accuracy.

SettingDescription
Enable Vector EmbeddingEnabled: Uses AI embedding models for semantic search. Finds results by meaning similarity rather than keyword matching.
Disabled: Uses traditional FULLTEXT search.
ProviderAuto Default - Uses the selected AI provider's embedding model
OpenAI - Uses OpenAI text-embedding models
Gemini - Uses Google embedding models
Note: Enabling vector embedding adds an extra API call per chat message, which may incur small additional costs.

MCP (Model Context Protocol) Server

Enables external AI tools like Claude Desktop, Cursor, and other MCP-compatible clients to access your knowledge base.

SettingDescription
Enable MCPEnabled: Exposes a REST API endpoint for external AI tools to search your knowledge base and retrieve site info.
Disabled: MCP endpoint is not accessible.
API KeyClick "Generate" to create a new API key. Displayed only once. Use as Bearer token in external tools.
Endpoint URLAuto-generated REST API URL. Enter this in your MCP client configuration.

Chat Settings

Bot Name

The assistant's name displayed in the chat window header and messages.

Default: Assistant

Customize to match your brand, e.g., "Support AI", "Customer Assistant", etc.

Bot Avatar

The icon displayed next to chat messages from the bot.

Default: Robot emoji

Enter an emoji directly or click "Upload Image" to select from the media library. Images are automatically resized.

Welcome Message

The first message displayed when a user opens the chatbot.

Default: Hello! How can I help you today?

Customize to match your site's purpose. When Response Language is set to "Auto-detect", you can configure per-language welcome messages.

Response Language

Controls which language the AI responds in.

OptionDescription
Site Language DefaultFollows WordPress language settings
Auto-detectDetects user input language and responds in the same language. Ideal for multilingual sites. Reveals per-language welcome message settings.
Specific LanguageAlways responds in the selected language (en, ja, zh, ko, es, fr, de, pt)

Per-Language Welcome Messages (Auto-detect mode only):

Set individual welcome messages for 13 languages: English, Japanese, Chinese, Korean, Spanish, French, German, Portuguese, Italian, Russian, Arabic, Thai, Vietnamese. The appropriate message displays based on browser language detection.

System Prompt

Instructions that define the AI's behavior, personality, and constraints. Sent with every conversation behind the scenes.

Default: Pre-configured with rules about accuracy, honesty, and avoiding fabrication.

Tip: Include your company name, services, topics to avoid, and desired tone. Use the "Reset to Default" button to restore the original prompt.

Context Prompts (Advanced)

Customize how knowledge base and site learning data is presented to the AI. The defaults work well for most use cases.

PromptDescription
Knowledge Exact MatchInstructions when an exact Q&A match is found. Directs AI to return the registered answer directly.
Knowledge Q&AInstructions for referencing Q&A-format knowledge base entries.
Site ContextInstructions for referencing crawler-indexed content.

Feature Prompts (Advanced)

Customize the behavior of regeneration, feedback, and summary features.

PromptDescription
Regenerate PromptInstructions for regenerating responses. Placeholders: {variation_number}, {forbidden_start}, {style}.
Feedback Good/Bad HeadersHeaders for showing the AI examples of positively/negatively rated responses.
Summary PromptInstructions for generating conversation summaries.

Message History Count

Number of previous messages sent to the AI as context. More messages = better context understanding but higher API costs.

Default: 10 | Range: 1 - 50

Recommended: 10-15 for general support. Reduce to 5 for cost savings, but the AI may lose track of earlier questions.

Max Tokens

Maximum output tokens per AI response. One token is approximately 4 English characters or 1 Japanese character.

Default: 1000 | Range: 100 - 4000

GPT-5 Auto-Multiplier: GPT-5 models consume more tokens for reasoning, so the configured value is automatically multiplied (default: 4x). Setting 1000 results in an effective limit of 4000 tokens.

Temperature (Creativity)

Controls response randomness. Lower values produce consistent responses; higher values produce diverse responses.

Default: 0.7 | Range: 0 - 2 (step 0.1)

RangeBehaviorBest For
0 - 0.3Highly consistent. Same question yields same answerFAQ, factual responses
0.5 - 0.8Balanced responsesGeneral customer support
0.9 - 1.5Creative, diverse responsesCreative content
1.5 - 2.0Highly random, unpredictableUsually not recommended

Feedback Buttons

Displays thumbs-up/thumbs-down buttons below each AI response.

Enabled: Users can rate responses. Admins can review feedback in the conversations panel for quality improvement.

Disabled: Hides feedback buttons.

Web Search

When the knowledge base doesn't have sufficient information, falls back to web search results as AI reference.

Enabled: Automatically performs web searches when knowledge base lacks relevant content.

Disabled: Responds using only knowledge base and site learning data.

Note: Enabling web search adds extra API calls and may increase costs.

Select the chat widget's visual theme. Six themes are available in the Free edition.

Primary Color

The main color used for the chat header, buttons, and badge.

Default: #007bff (blue)

Use the color picker to select any color. Selecting a theme automatically sets a matching color.

Badge Position

Screen position of the floating chat badge button.

OptionDescription
Bottom-Right DefaultMost common placement. Suitable for most sites
Bottom-LeftWhen other elements occupy the bottom-right
Top-RightLess common, available for specific layouts

Badge Margins

Distance from screen edges in pixels.

Right Margin Default: 20px | Bottom Margin Default: 20px

Adjust to avoid overlapping with other floating elements like cookie banners.

Markdown Rendering

Converts Markdown syntax in AI responses to rich text.

Enabled Default: **bold**, *italic*, lists, code blocks, and links render as formatted text.

Disabled: Markdown symbols display as plain text.

Mobile Display

Controls whether the chat widget appears on mobile devices.

Enabled Default: Chatbot visible on all devices.

Disabled: Chatbot visible on desktop only. Use when prioritizing mobile performance.

Cross-Site Embed

Embed the chatbot on non-WordPress sites using provided code snippets.

Script Embed: Floating badge. Insert the <script> tag before </body>.

iframe Embed: Fixed-size chat window. Place the <iframe> tag anywhere on your page.

reCAPTCHA Settings

Enable reCAPTCHA v3

Enables Google reCAPTCHA v3 bot protection. Version 3 scores requests in the background without user interaction.

Enabled: Validates reCAPTCHA scores on message submission, blocking automated bot messages.

Disabled: No reCAPTCHA validation.

Site Key / Secret Key

Obtain these from the Google reCAPTCHA Admin Console.

Site Key: Public key used on the frontend.

Secret Key: Server-side verification key. Encrypted and masked after saving.

Score Threshold

Requests scoring below this threshold are blocked.

Default: 0.5 | Range: 0.0 - 1.0

  • 0.1 - 0.3: Lenient. Allows most requests
  • 0.5: Standard. Recommended
  • 0.7 - 0.9: Strict. May block legitimate users

Fail Mode

Behavior when reCAPTCHA verification itself fails (e.g., network errors).

OptionDescription
Open (Allow) DefaultAllows chat when verification fails. Prioritizes availability
Closed (Block)Blocks chat when verification fails. Prioritizes security

Advanced Settings

Save Conversation History

Enabled Default: Conversations are persisted to the database. Viewable in the admin "Conversations" page. Conversations restore on page reload.

Disabled: Conversations exist only during the session and are lost when the user navigates away. Use for privacy-first deployments.

Retention Period

Days to keep conversation history. Older conversations are automatically deleted.

Default: 90 days | Set to 0 for unlimited retention

Delete Data on Uninstall

Enabled: Removes all plugin data (settings, conversations, knowledge base, indexes) when uninstalled.

Disabled Default: Preserves data for potential reinstallation.

WP Consent API Strict Mode

Integrates with the WP Consent API plugin for GDPR cookie consent.

Enabled: Requires active consent before using localStorage. Session IDs and chat history are not stored in the browser without consent.

Disabled Default: Uses localStorage without consent verification.

Rate Limiting

Limits message frequency per user to prevent API cost spikes and spam.

SettingDefaultDescription
Max Requests20Maximum messages allowed within the time window. Set to 0 for unlimited.
Time Window1 hourRate limit period. Options: 1min, 5min, 10min, 30min, 1hr, 3hr, 6hr, 12hr, 1 day
Tip: Pro edition offers two-tier rate limiting (burst + sustained) with per-minute and per-hour limits.

Proxy Settings

Configure IP detection when using Cloudflare or load balancers.

SettingDescription
Trust Cloudflare IPUses the CF-Connecting-IP header for real client IP. Enable if using Cloudflare.
Trust Proxy IPUses the X-Forwarded-For header for real client IP. Enable if using a reverse proxy.
Warning: Enabling these settings inappropriately may create IP spoofing risks. Configure according to your infrastructure.

Site Learning (Crawler)

Enable Crawler

Automatically crawls your WordPress content and indexes it as AI reference knowledge.

Enabled Default: Periodically crawls specified post types, splits content into chunks, and stores in the search index. Relevant content is automatically included in AI responses.

Disabled: No crawling. Use only manually registered knowledge base entries.

Post Types

Which WordPress post types to crawl.

Default: All - Crawls all post types (posts, pages, custom post types). Can select specific types.

Crawl Interval

OptionDescription
Daily DefaultCrawls once per day. Best for frequently updated sites
WeeklyCrawls once per week. Best for stable content sites

Manual crawl is also available from the Dashboard's "Site Learning" section.

Chunk Size

Characters per content chunk when splitting crawled content.

Default: 1000 characters

  • Smaller (500-800): More precise search, but larger index
  • Larger (1500-2000): Broader context per chunk, but potentially less precise results

Max Results

Maximum number of context chunks sent to the AI per user query.

Default: 3

  • Fewer (1-2): Saves API costs but may miss relevant information
  • More (5-10): Better context coverage but higher API costs

Knowledge Base

Overview

The knowledge base is a collection of Q&A data the AI uses to provide accurate answers. Unlike site learning (crawler), manually registered answers are prioritized.

Free Edition Limit: Up to 20 knowledge base entries. Pro edition supports unlimited entries.

Add Q&A

Add question-answer pairs manually from the admin "Knowledge" page.

  • Question: The question users might ask
  • Answer: The exact answer the AI should provide
  • Category: Optional label for organization
Tip: Register multiple phrasings of the same question to improve recognition accuracy.

CSV Import

Bulk import knowledge base entries from a CSV file. Format:

question,answer,category
"What are your prices?","Plans start at $10/month.","Pricing"
"Business hours?","Monday to Friday, 9 AM to 6 PM.","Hours"

Conversations

Conversation List

View all chat conversations in the admin "Conversations" page. Shows session ID, start time, message count, and status for each conversation.

Message View

Click a conversation to view the chronological exchange between user and AI. Feedback ratings (thumbs up/down) are also displayed.

CSV/JSON export is available in the Pro edition. PRO
